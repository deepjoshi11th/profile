🐛 Overconfident First Fix
Fixed the issue... without having the issue.

It was my first task of my career during a critical release week...

I fixed what I saw, wrote tests that passed, and pushed. But that added new issue.

commit a1c3d1d - fixed bug (not really)
Author: Deep 

    Fixed the critical issue… on my machine.
    Wrote unit tests that validated my code change.

    Reviewed but skipped confirming with teammates(needful at that time).

Lessons:
  - Confidence is good — confirmation is better.
  - Ask for help more often.
  - Reproduce the bug before fixing it.
🖼️ Extension Error on Help Page
Image was updated, but not as expected — a quiet rename caused a loud miss.

During the final leg of a release, a help document image silently failed to load. I had updated it — but changed the file extension from .jpg to .jpeg.

The code was fine, the logic untouched, but my assumption went unchecked.

The image was broken — and it was caught by someone senior, making the moment heavier than it technically deserved.

commit e913ff2 - updated help logo
Author: Deep 

    Replaced image on help landing page.
    But changed its extension from .jpg → .jpeg
    Didn't verify in the actual rendered view.
    Broken image spotted by release manager.

Lessons:
  - Even the smallest changes deserve real validation.
  - Having many valid extensions isn't the same as following a standard.
  - Reviewers miss things. Build systems that don't.
  - Don't overweigh a bug just because a big name spotted it — the lesson stands on its own.

🔐 Shared a Password by Email
The sandbox was safe — but the habit wasn’t.

I was sharing test credentials with a vendor, like I had the previous week.

But this time, the word “password” in my email triggered an automatic escalation — and within hours, I had a 10-step compliance form in my inbox from the VP.

No data leaked. No machine was breached. But the process flagged a pattern that needed to be stopped — and I was that pattern.

commit f4cc3s5 - emailed password in safe context
Author: Deep 

    Gave credentials for security testing to external vendor.
    Machines were sandboxed. Network was local.
    Word "password" in email auto-triggered VP escalation.

Lessons:
  - Security policies exist for edge cases, not everyday ones.
  - The habit, not the context, defines the risk.
  - Escalations measure behavior risk, not just breach severity.
  - “No harm done” doesn’t mean “no harm possible.”

Reflection:
  - It could’ve been anyone — but I owned the product.
  - I calmly completed all 10 remediation steps for the team.
  - Security is everyone’s job — especially when it feels optional.
